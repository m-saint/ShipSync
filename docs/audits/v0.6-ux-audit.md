### 5. Error Prevention â€” PASS

- `setShipSceneCondition` validates that the target ship still exists (player or scene) before committing; otherwise the call is a silent no-op (`workspace.svelte.js:1722`â€“`:1724`). Verified by `workspace.test.js` `setShipSceneCondition is a no-op for unknown ship ids`.
- `loadFile.js`'s `sanitizePersistentConditions` defends against junk in older save files: non-array `conditions` fields reset to `[]` with a warn-level issue; unknown ids are dropped (count surfaced in the warning); duplicates are deduped before reaching application state (`loadFile.js`'s `sanitizePersistentConditions`, exercised by `loadFile.test.js` "drops unknown persistent condition ids with a warn issue" and "deduplicates persistent conditions").
- Player-ship `removeShip` now matches `removeSceneShip`'s scene-state cleanup. A latent bug was patched: removing a player ship that previously held the Weather Gage left `weatherGageHolderId` pointing at a deleted id; the cleanup now clears it (`workspace.svelte.js` `removeShip`).
- The mixed-persistence model itself is an error-prevention move: scene chips can't accidentally be saved into the ship file, and persistent chips can't accidentally be reset by reloading the workspace, because the storage paths are physically separate (`scene.shipConditions[shipId]` vs `ship.conditions[]`). The two paths can't collide.
- Flag collapse is a prevention measure for a more subtle error class: a ship with seven flags pushing the FlagsSection ten screens long would invite the player to lose track of which flag they're flying. Promoting the `flyingId` flag to the always-visible list (even if its array position would otherwise hide it) makes "what colors am I flying right now?" recognizable at a glance (`FlagsSection.svelte:55`â€“`:64`). The trade-off is a possible visual order gap when the flying flag was added late; this is documented inline (`:42`â€“`:44`).

### 8. Aesthetic and Minimalist Design â€” PASS

- `ConditionsSection` is one card with one h3 header, two row groups, no decorative chrome â€” the divider between persistent and scene rows is a 1px `border-t border-surface-200` (`ConditionsSection.svelte:107`).

- The four scene chips are a deliberate floor â€” Heeling / In Irons / Crossing the T / Becalmed cover the most-cited scene conditions in the rules.
