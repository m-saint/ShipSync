## Findings by Heuristic

### 1. Visibility of System Status ‚Äî PARTIAL ‚Üí addressed in v0.4.1

- [x] Round, phase, wind, and weather-gage state are visible at all times in the right rail
- [x] Pursuit state communicates "you're being caught" / "you're getting away" through copy
- [x] Every scene mutation logs a human Captain's Log line (`Pursuit: gap 3 ‚Üí 2.`, `Round 1 ‚Üí 2.`, `Boarded by Crimson Maw.`)
- [ ] **F1.1**: A fresh pursuit opened at gap=0, which read as the "Caught ‚Äî the chase becomes a fight" alarm state on the very first frame. The user had to step the gap up to escape that visual. *Resolved in v0.4.1 by defaulting `makePursuit().gap` to 5.*
- [ ] **F1.2**: The crimson "Caught" warning text was the only visual cue in the alarm state ‚Äî the Gap stepper itself stayed neutral. Same gap on the side text didn't have the prominence the situation called for. *Resolved in v0.4.1 by extending `NumberStepper` with `crimson` and `sea` tones and piping the gap into the stepper, not just the side text.*

### 2. Match Between System and the Real World ‚Äî PASS

Scene copy is in nautical voice in headlines ("Open seas.", "No chase underway.", "Drifting / in irons") and neutral in form labels ("Pursuer", "Quarry", "Gap", "Escape condition"), holding the v0.1/v0.2 tone split. The "Boarded by" field is a noun the user already uses at the table; no rule reference required.

### 3. User Control and Freedom ‚Äî PASS

Every scene mutator goes through `commit()`, so every step is undoable through the existing toolbar / `‚åòZ`. Pursuit can be toggled off and on without confirmation (no destructive payload ‚Äî gap and pointers reset by design). Removing a sighted scene ship is a single click that's instantly undoable.

### 4. Consistency and Standards ‚Äî PARTIAL ‚Üí addressed in v0.4.1

- [x] Reuses `Field`, `RuleTooltip`, `NumberStepper`, and the right-rail typography scale
- [x] Bridge Crew collapse toggle on `OfficerRoster` honors the same disclosure pattern used elsewhere
- [ ] **F4.1**: Round counter used a hand-rolled pair of `‚àí`/`+` buttons (24√ó24, custom border) instead of the `NumberStepper` (32√ó32, shared border). The stepper rhythm in the right rail broke between Round and Gap. *Resolved in v0.4.1 by adding a `setSceneRound(value)` mutator, refactoring `advanceRound(delta)` through it, and replacing the custom buttons with `NumberStepper`.*
- [ ] **F4.2**: Size / Mobility / Disposition / Phase selects rendered raw lowercase enum values (`tiny`, `balanced`, `hostile`, `idle`) in the option popups. Tailwind's `capitalize` class only paints the closed trigger, not the OS-native popup. The same enum looked title-cased when closed and lowercase when open. *Resolved in v0.4.1 by introducing `SHIP_SIZE_LABELS`, `MOBILITY_LABELS`, `DISPOSITION_LABELS`, `PHASE_LABELS` in `rules.js` and routing every visible string through them.*

### 5. Error Prevention ‚Äî PASS

Scene mutators clamp every numeric field (round ‚â• 0, gap ‚â• 0, fires ‚â• 0, hp ‚â• 0, explosionDC ‚â• 1). Removing a scene ship cleans up dangling references in `weatherGageHolderId` and `pursuit.{pursuerId,quarryId}` so the workspace can never reach an inconsistent state. `addSceneShip` enforces a unique id; `setSceneShip` short-circuits no-op patches.

### 6. Recognition Rather Than Recall ‚Äî PASS

The right rail surfaces every piece of scene state at once ‚Äî no hidden tabs, no separate "scene editor" modal. The `allShipsForReference` derivation lets the Pursuer / Quarry / Weather-gage holder dropdowns show every ship in the workspace (loaded *and* sighted) by label, so the user picks from a list rather than typing an id.

### 7. Flexibility and Efficiency of Use ‚Äî PASS

‚åò‚áßS Save All shortcut now lives next to ‚åòS per-ship save (v0.3 deferred A2). Bridge Crew collapses to free vertical space (v0.3 deferred A1). Grub / Grog tint amber at zero (v0.3 deferred A3). Pursuit's `Begin pursuit` / `End pursuit` toggle is a single click, no modal.

### 8. Aesthetic and Minimalist Design ‚Äî PASS

The right-rail Scene panel uses internal scrolling (`flex-1 min-h-0 overflow-y-auto`) so the Captain's Log below stays anchored even when the scene gets crowded with sighted vessels. Each subsection has a single small headline and no decorative chrome. The v0.3 PARTIAL finding ("Bridge Crew at full strength is a long block") is closed by the new collapse toggle.

### 9. Help Users Recognize, Diagnose, and Recover from Errors ‚Äî PASS

Pursuit "Caught" copy names what's happening *and* what it costs ("the chase becomes a fight"). Removing a ship that holds the weather gage transparently transfers the "drifting / in irons" label to the slot ‚Äî no error toast, no broken reference. Saved a v0.3 ship file with a missing `boardedBy` field still loads cleanly thanks to `repairShip`'s default.

### 10. Help and Documentation ‚Äî PARTIAL ‚Üí addressed in v0.4.1

- [x] Each scene subsection has a `RuleTooltip` hint that explains the field in sailor-language
- [x] Empty states for sighted ships ("Open seas. No vessels sighted. Add a sighting when one comes within view.") teach the user what the section is for
- [ ] **F10.1**: There was no in-UI signal that scene state lives only in the autosave snapshot ‚Äî a user editing the round counter and then saving the focused ship file might reasonably expect the round to ride along in the file. *Resolved in v0.4.1 by adding an "autosave only" subhead to the ScenePanel header with a `RuleTooltip` that explains the persistence asymmetry.*
